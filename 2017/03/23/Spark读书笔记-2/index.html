键值对操作

1.创建pair RDD
val pair = rdd.map(x => (x.split(“ “)[0],x)) 函数需要返回二元组

2.pair RDD的转换操作
聚合：reduceByKey combineByKey
连接：join 内连接，只输出两个RDD中都有的键值
leftOuterJoin() 左连接 以第一个RDD中的键为输出键，对应两个RDD中的值
rightOuterJoin() 右连接 以第二个RDD中的键为输出键
分组：groupByKey()
排序：sortByKey 自定义排序 重写compare

3.pair RDD的行动操作
countByKey
collectAsMap
lookup(key)

数据分区

通过控制RDD分区来减少通信开销；当数据集需要多次进行基于键的操作（比如join，sort，group）时，分区能有帮助
数据持久化到集群后，按照被读取时的来源来分布，所以无法获取某个键对应记录位于哪个节点上；在进行基于键的操作时会对所有键的记录进行操作；

自定义分区：
（1）在创建RDD时进行分区并持久化（否则下次用到这个RDD还会进行分区）
（2）partitionBy()：转化操作，获取一个已经分区的新RDD，Spark有它的分区信息；在进行可利用分区信息的操作时（join,sort,group），可减少数据的网络传输，提高程序运行速度。
（3）partitionedRDD.partitioner 获取RDD分区方式
（4）尽量利用父RDD的分区信息（比如用mapValue代替map），减少开销
（5）自定义分区器： 继承partitioner类，重新方法：
numPartitions:分区数
getPartition（key：Any）：Int ：返回给定键的分区编号
equals()：判断两个分区方式是否相同
