Spark读书笔记(1)

Mar 23, 2017 | Hits

Spark相比于MapReduce的优势

1.支持更多计算模型，交互式查询、流计算（动态数据，实时计算）
2.能够在内存中计算，运算速度快
3.更适合重复读写数据，迭代计算

Spark组件

1.Spark core
Spark基本功能，任务调度、内存管理、错误修复，RDD API定义

2.Spark SQL
操作结构化数据

3.Spark Streaming
对实时数据进行流计算

4.Mlib
机器学习程序库

5.Graphx
并行图计算，支持各种图操作，图算法

Spark核心概念

1.sc:Spark的驱动器程序通过SparkContext对象来访问Spark，Spark shell启动时自动创建SparkContext对象，sc变量，调用sc的方法创建RDD。

2.RDD：弹性分布式数据集，分布式的元素集合，RDD的数据会被Spark自动分发到集群上进行并行操作

RDD编程

创建RDD的方法：
1.应用外部文件系统（比如HDFS）的数据集创建RDD
val lines = sc.textFile(“filepath”)

2.并行化一个已经存在于驱动程序中的集合（比如list，set）创建RDD
val rdd = sc.parallelize(List(1,2,3))
val rdd2 = sc.makeRDD(List(1,2,3))

RDD两种类型的操作：
1.转化(transformation)，由原RDD生成一个新的RDD
val flines = lines.filter(line=>line.contain(“scala”))
*filter不会改变原来的RDD，而是生成全新的RDD
常用转换操作：
map 从原RDD映射得到新的RDD（一生一）
filter 从原RDD筛选得打新的RDD
flatMap 从原RDD的一个元素映射得到多个元素（一生多），返回迭代器
伪集合操作：distinct union intersection substract cartesian

2.行动(action)，对RDD计算结果，并将结果返回驱动器程序或存储到外部存储设备
val fi = flines.first()
常用action操作： reduce collect（获取整个数据集） count take（获取RDD中少量元素）
aggregate:提供初始值，并返回不同类型的函数
*Spark对于新定义的RDD的计算是惰性的，即定义RDD后并不会马上执行和读取数据，只有在第一次行动操作用到时才真正执行并读取数据到内存；
所以转换操作并不是马上执行，只是记录此次操作，等到对RDD进行action操作时才真正启动计算。

RDD持久化到内存：
Spark默认每次对RDD进行行动操作时都会重新计算；
持久化即将RDD的内容保存在（集群的）内存或磁盘中，再次执行行动操作时可以重用这些数据，缩短执行时间。
持久化级别：MEMORY_ONLY(只存在内存中) DISK_ONLY（只存在磁盘中）
持久化操作：RDD.persist(StorageLevel.MEMORY_ONLY) RDD.cache()
LRU策略：内存放不下数据时，将最近最少使用的数据移出内存

RDD类型转换
不同的RDD类型：基本RDD类，各种封装类（DoubleRDDFunctions:数值RDD；PairRDDFunctions:键值对RDD）
scala采用隐式转换，当Scala编译器进行类型匹配时，如果找不到合适的候选，那么隐式转化提供了另外一种途径来告诉编译器如何将当前的类型转换成预期类型。
